Concept of Git

Git is a version control system for tracking changes in computer files. it is used for coordinating work among several people on a project and tracking progress overtime.

Git is a widely used version control system (VSC) that lets you track of all the modifications you make to your code. Git is a distributed VCS, this means that every collaborator of the project will have a history of the changes made on their local machine.

GitHub

Github on the other is a widely used platform for version control that used Git at its core. It lets you host the remote version of your project from where all the collaborators can have access to it. Not just your own team member,but any member of Github can contribute to your code(that is if you choose to accept the changes made). Github is like a social platform where you can find a plethora of open-source projects with their codes. All the new and emerging technologies can be found on this platform Github.com

Terminologies in Git

  • Repository: Repo is a folder ths contain all the project files that the history of the versions made to each.

    There are two repositories of your project that you will work throughout the lifetime of your project.

    1. Remote repo:- Its contains your project that can be accessed from anywhere and by anyone. this is the remote repository live on github and anybody can access it.
    2. Local Repo:- This is a copy of the remote repo that resides on your local machine. All the changes you make will be savedin your local repo. Fellow developers will not be able to see them yet untill it is push to the remote repo.

  • Cloning: means creating a copy of the remote repo on your local machine. Now you can make changes to the project on your local machine.

Basic Git commands

  • Git Config-- configure the username and email address
  • Git init-- Initiaze an empty Git repo
  • Git add-- Add the content to the staging area
  • Git diff-- View the changes made to the file
  • Git commit-- Makes sure that the changes are saved to the local repo
  • Git reset-- Undo local changes to the state of a Git repo
  • Git status-- Display the state of the working diectory and staging area
  • Git merge-- Merge a branch into an active branch
  • Git Push-- Upload content from local repo to a remote repo
  • Git Pull-- Fetch and merge changes from the remote repo

Beginner Step to using Git command prompt

    Step 1
  1. Make sure terminal has right project directory

    2. Step 2
  2. Initialise a repository using git init command

    3. Step 3
  3. Configure Git: Make git know who make the changes by using git config command

    4. Step 4
  4. Add file to project directory using git add Adding files are in a place called staging

    Staging area is an intermediate place between working directory and local git repo where any changes that are made can be reviewed before actually been commited them to the repo.


    5. Step 4
  5. After the staging area, take a snapshot of all the changes made, which are reflected in the staging area, and save them in the Git repo using git command -m 'commit message'

    NOTE: The whole point of version control is to keep a record of the changes that were made. Do this using the git log command it gives a complete view of all the commits that were made in reverse chronological order.


    6. Step 5
  6. Create a new branch. Branch allow to move back and forth between 'states' of a project. when one create a new branch, git keep track of which commit the branch "branched off of", so it knows the history behind all the files.

    Let's says you are on the master branch and want to create a new branch to develop your web page. Run: git checkout -b This command will automatically create a new branch and then 'check you out' on it i .e git will move you to that branch, off of the master branch.

    Use 'git branch' command to confirm that yout branch was created

    7. Step 6
  7. Create a new repository on Github: This require your log in and creating a new repo on github home page.

    8. Step 7
  8. Push a branch to Github: Push the commit in your branch to your new GitHub repo. Tis allows other people to see the changes you've made. This is done by using the command key git push -u origin yourbranchname

Other commands include the git pull, git merge including advanced git attribute.

Cloning a Github repository

We can create our own local repo and push it to Github. if you aren't the on that created the repo.One would have to make a copy on local machine. Once a project has been uploaded onto the remote repository,developers can get their own copy of this repo using the git clone command. To clone to local machine,go to terminal and provide the URL in the following command git clone URL

Branching and merging: Branches let you work on new features or bug fixes of the main project code that is presenton the master branch.
Do not pull directly to the master branch (main project branch) especially when working with group of dev. Branches let you experiment with new features or idea and even let you create a new branches in parallel to experiment with different features. Create new branch using git branch

Once confident that the code is working well, integrate it with master branch using the git merge command
To merge to master when sure there is no more bugs: git merge branch name -m "messages"

Pull using Git:
Pull: Lets you fetch any updates from the remote repo and merge it with the local repo, using pull command git pull orgin branch-name
